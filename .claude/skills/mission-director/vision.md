# The Vision: What We're Building Toward

## The Three-Phase Promise

**REPLICATE → ENHANCE → AI-NATIVE**

1. **REPLICATE** — Copy the contractor's existing website. Same brand, same feel, same
   content, same identity. A homeowner landing on this site should think "oh, they
   redesigned" — not "this is a different company."

2. **ENHANCE** — Bring it to 2026 premium standards. Beautiful design with intentional
   typography (not Inter/Roboto), orchestrated animations (stagger reveals, hover lifts),
   atmospheric backgrounds (gradients, grain textures), and polished micro-interactions.
   The site should feel like a senior designer spent a week on it.

3. **AI-NATIVE** — Embed ConversionOS underneath. AI quote engine, design visualizer,
   chat widget, voice receptionist, full admin dashboard. The contractor gets a website
   that actively captures and qualifies leads — not just a brochure.

## What "Premium" Means (Not Template-Like)

**Template-like (BAD):**
- Every section has identical `py-16 px-4` spacing
- Inter or Roboto font everywhere at weight 400 and 600
- Cards are white rectangles with `rounded-lg shadow-sm` (all identical)
- Purple gradient backgrounds (the signature AI slop tell)
- No hover effects, no scroll animations, no visual rhythm
- Could be any company — nothing site-specific

**Premium (GOOD):**
- Spacing rhythm varies intentionally: hero is generous (100vh), services are tight (py-20), CTA is compact (py-12)
- Typography uses weight extremes: 200 for decorative, 800 for headlines, 3x+ size jumps
- Cards have unique treatments: subtle shadows, border-radius matching the original, hover lift + shadow expansion
- Colours follow the original's palette: primary on CTAs/accents only, not large backgrounds
- Orchestrated page load: hero fades in, then headline, then CTA (staggered, not simultaneous)
- Scroll reveals via IntersectionObserver, cards stagger in grid sections
- Atmospheric backgrounds: layered gradients, warm neutrals, dark overlays with grain

## The Design Language Document (Our Secret Weapon)

The Design Language captures the visual DNA of the original site. It tells Codex:
- "This site uses burgundy sparingly — only CTAs and hover states"
- "Section padding is 96px desktop, 48px mobile"
- "Cards have 12px radius, shadow 0 4px 24px rgba(0,0,0,0.08)"
- "Typography is serif headings (Playfair Display) + sans-serif body"

Without this, Codex generates structurally correct but visually generic code.
With it, Codex generates code that matches the original's aesthetic DNA AND
incorporates premium upgrades.
